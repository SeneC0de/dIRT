# KICKOFF — starting a Director session

Open Cowork in a project repo (any folder containing `project.json`, including subfolders) and as your first message paste:

```
Run `dcli kickoff` and follow the instructions it returns.
```

That's it. The `dcli kickoff` command prints the absolute paths to the canonical doctrine files and the startup ritual. Claude reads the doctrine, adopts the project's character + personality from `project.json`, and you're off.

Prerequisite (one-time per machine): the `director-pattern` repo root on PATH so `dcli` resolves. See README §Setup.
